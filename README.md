6. Краткий список реализованных проверок.

Позитивные сценарии - авторизация:

1) Получение токена (auth/login) | Ожидаем 200 ОК
2) Получение профиля (auth/me) | Ожидаем 200 ОК

Позитивные сценарии корзина:

3) Получение корзины через userId (/carts/user/{{user_id}}) | Ожидаем 200 ОК
4) Получение корзины через cartId (/carts/{{cartId}} ) | Ожидаем 200 ОК
5) Добавить новую корзину через userId (/carts/add) | Ожидаем 201 Created
6) Замена существующих значений в корзине (/carts/{{cartId}}  (put)) | Ожидаем 200 ОК
7) Добавление значений в существующую корзину (/carts/{{cartId}}  (patch)) | Ожидаем 200 ОК
8) Удаление существующей корзины (/carts/{{cartId}}) | Ожидаем 200 ОК

Негативные сценарии - авторизация:

9) Авторизация с несуществующим логопасс(auth/login) | Ожидаем 400 Bad Request
10) Получение профиля без токена (auth/me) | Ожидаем 401  Unauthorized
11) Получение профиля с неверным токеном (auth/me) | Ожидаем 401 Unauthorized

Негативные сценарии - корзина:

12) Получение корзины несуществующего пользователя (/carts/user/{{user_id}}) |  Ожидаем 404 Bad Request

13) Валидация полей при добавлении новой корзины (/carts/add ) |  Ожидаем 400 Bad Request
*Ручка будет отдавать 201 Created, так как dummyjson_collection мокирует данные и не проводит проверку на валидацию самих полей.

14) Передача неверного JSON при добавлении корзины(/carts/add) |  Ожидаем 404 Bad Request
